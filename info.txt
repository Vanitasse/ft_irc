Exemples de support information

    CHANTYPES: Types de canaux supportés par le serveur.
        Exemple: CHANTYPES=#&
        Signifie que les canaux peuvent commencer par # ou &.

    PREFIX: Les préfixes de statut des utilisateurs dans les canaux et leur correspondance avec les modes.
        Exemple: PREFIX=(ov)@+
        o pour les opérateurs de canal avec le préfixe @, v pour les utilisateurs avec la voix avec le préfixe +.

    MODES: Nombre maximum de modes qui peuvent être changés dans une seule commande MODE.
        Exemple: MODES=4
        Vous pouvez changer jusqu'à 4 modes en une seule commande.

    CHANTYPES: Types de canaux supportés par le serveur.
        Exemple: CHANTYPES=#
        Signifie que les canaux commencent par #.

    CHANLIMIT: Limites sur le nombre de canaux qu'un utilisateur peut rejoindre.
        Exemple: CHANLIMIT=#&:10
        Un utilisateur peut rejoindre jusqu'à 10 canaux de type # et &.

    NICKLEN: Longueur maximale des pseudonymes.
        Exemple: NICKLEN=30
        Les pseudonymes peuvent avoir jusqu'à 30 caractères.

    TOPICLEN: Longueur maximale du sujet d'un canal.
        Exemple: TOPICLEN=307
        Les sujets de canal peuvent avoir jusqu'à 307 caractères.

    AWAYLEN: Longueur maximale du message d'absence.
        Exemple: AWAYLEN=160
        Les messages d'absence peuvent avoir jusqu'à 160 caractères.

    MAXLIST: Limite sur le nombre d'éléments dans certaines listes, comme les bans.
        Exemple: MAXLIST=b:25,eI:50
        Vous pouvez avoir jusqu'à 25 bans (b) et 50 exceptions (eI).

    NETWORK: Nom du réseau IRC.
        Exemple: NETWORK=Freenode
        Le nom du réseau est Freenode.

    CASEMAPPING: Le type de mappage des caractères pour les comparaisons insensibles à la casse.
        Exemple: CASEMAPPING=rfc1459
        Le mappage des caractères est basé sur la norme RFC 1459.

    EXCEPTS: Indique si le serveur supporte les exceptions de ban (e).
        Exemple: EXCEPTS=e
        Le serveur supporte les exceptions de ban.

    INVEX: Indique si le serveur supporte les exceptions d'invitation (I).
        Exemple: INVEX=I
        Le serveur supporte les exceptions d'invitation.

    CHARSET: Jeu de caractères utilisé par le serveur.
        Exemple: CHARSET=utf-8
        Le serveur utilise UTF-8.

    ELIST: Fonctionnalités supplémentaires pour la commande LIST.
        Exemple: ELIST=CTU
        Supporte les options C (comparaison insensible à la casse), T (informations sur le sujet), U (informations sur les utilisateurs).
